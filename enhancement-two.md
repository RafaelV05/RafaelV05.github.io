Enhancement Two: Algorithms and Data Structures

The artifact I selected for this enhancement is my Travlr Getaways full-stack web application, which I originally created in CS 465 Full Stack Development. The application allows users to browse travel packages, create accounts, make reservations, and manage travel information. It uses a Node.js and Express backend, MongoDB for data storage, and an Angular administrative interface.
I chose this project because it is one of the largest and most complete applications I built during the program. It contains multiple layers that work together, including the user interface, API routes, controllers, and database operations. Since the application already had a solid foundation, it provided a good opportunity to improve how data is processed and displayed without having to redesign the entire system.

I selected this artifact because it demonstrates several areas of software development, including frontend development, backend development, database management, and API design. For this enhancement, I focused specifically on improving how trip data is processed before it is displayed to users.
Originally, the application simply retrieved all available trips from MongoDB and displayed them in the Angular interface. While this worked for a smaller dataset, it would become less practical as the number of travel packages increased. Users would have to scroll through every available trip to find what they were looking for.
To improve the application, I added searching, filtering, and sorting functionality. Users can now search for trips using keywords, filter results by resort, maximum price, and trip length, and sort trips by name, price, duration, or start date. These features required processing collections of Trip objects stored in arrays before presenting the results to the user.
The enhancement demonstrates the use of common algorithms and data structures. Filtering operations iterate through arrays and evaluate specific criteria to determine which trips should be included in the results. Sorting operations organize the remaining trips based on user-selected attributes. Search functionality compares user input against trip properties to locate relevant matches. While these are relatively simple algorithms, they significantly improve the usability and scalability of the application.
I also made a design decision to keep most of the processing logic within the backend rather than placing everything in the Angular frontend. The Angular components collect the search and filter criteria from the user and send requests through service classes to the Express API. The controller layer processes the data and returns the appropriate results. Keeping this logic centralized helps improve maintainability and creates a cleaner separation between the presentation layer and business logic.
This enhancement showcases my ability to work with data structures, apply algorithmic thinking to solve practical problems, and improve an existing application without changing its overall architecture.

My goal was to improve how the application handled and processed trip data, and I successfully accomplished that through the implementation of search, filtering, and sorting features.
This enhancement aligns most closely with Outcome Three because it focuses on improving an existing computing solution through the use of algorithms and data structures. The enhancement required analyzing how trip information was stored and processed, then implementing a solution that made the data more useful to end users.
It also supports Outcome Four because I applied software engineering principles to design, implement, test, and integrate new functionality into an existing full-stack application. The enhancement improved both usability and maintainability while preserving the original system architecture.
At this time, I do not have any major updates to my outcome-coverage plans. This enhancement continues to support the outcomes I identified in Module One and demonstrates growth in my ability to apply technical concepts within a real-world software project.

One of the biggest things I learned during this enhancement is how much of a difference relatively simple algorithms can make in an application. The original version worked correctly, but users had no easy way to find specific trips or organize the results. Adding search, filtering, and sorting made the application feel much more complete and practical.
I also gained a better understanding of how the different layers of a full-stack application communicate with one another. User input is collected in Angular components, passed through service classes, sent to the Express API, processed by controllers, and then returned to the frontend for display. Working through this process reinforced the importance of keeping responsibilities separated across different parts of the application.
One challenge I faced was making sure that all of the new features worked correctly together. The enhancement required modifications to Angular components, service classes, backend controllers, and API routes. Because several files were involved, it was important to carefully test the flow of data throughout the application.
To validate the enhancement, I tested different combinations of search terms, resort filters, price limits, trip lengths, and sorting options. I verified that the correct results were returned and that the application continued to behave as expected when multiple filters were applied at the same time. This testing process helped identify minor issues and ensured the functionality was working correctly before completion.
Another lesson I learned was the importance of designing with future growth in mind. While the current dataset is relatively small, implementing structured search, filtering, and sorting logic now creates a foundation that can support larger amounts of data in the future. It also makes the application easier to maintain because the processing logic is organized and separated from the user interface.
Overall, this enhancement strengthened my understanding of algorithms, data structures, software architecture, and testing practices. It showed me how relatively small improvements to data processing can have a significant impact on both user experience and long-term scalability.
